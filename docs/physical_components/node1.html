Node 1: Access Control System

The Access Control System, utilizes the Raspberry Pi Pico W to implement RFID-based access control, enhancing security by allowing or denying entry based on authorized credentials. This system integrates with IoT platforms to monitor access in real-time, making it ideal for secure areas in educational, corporate, or residential environments. Its role within the cyber-physical system framework is to ensure a secure and manageable entry point that leverages cloud connectivity for efficient access logging and control.

Components Overview​

Controller: Raspberry Pi Pico W​

The Raspberry Pi Pico W is a robust microcontroller designed for projects requiring low-power consumption and cost-effectiveness. It integrates a dual-core ARM Cortex-M0+ CPU with a peak frequency of 133 MHz, and is equipped with 264KB of RAM and 2MB of Flash memory. With on-board wireless capabilities including 2.4GHz Wi-Fi and Bluetooth, it supports a wide range of applications from IoT to automation and sensor networks. The array of GPIO pins and analog inputs makes it versatile for interfacing with various sensors and actuators.

Technical Specifications and Features​

  • Microcontroller: RP2040, with 2MB of flash memory.
  • Wireless Interfaces: Single-band 2.4GHz wireless interfaces supporting 802.11n, Bluetooth 5.2, LE Central and Peripheral roles, and Classic Bluetooth.
  • Connectivity:
    • Micro USB B port for power, data, and programming.
    • 40-pin DIP-style PCB with edge castellations.
    • 26 multi-function 3.3V GPIO pins, 23 digital-only and three ADC capable.
  • Programming and Debugging:
    • 3-pin Arm serial wire debug (SWD) port.
  • Power Supply:
    • Flexible power supply options via micro USB, external sources, or batteries.
  • Cost and Availability: High quality and availability at a low cost.
  • Software and Documentation: Comprehensive SDK, with various software examples and detailed documentation.

Key Features​

  • Processor: Dual-core Cortex M0+ at up to 133MHz, with an on-chip PLL for variable core frequency.
  • Memory:
    • 264kB multi-bank high-performance SRAM.
    • External Quad-SPI flash with XIP and 16kB on-chip cache.
  • USB Support: On-board USB1.1 (device or host).
  • GPIO:
    • 30 multi-function general purpose I/O (4 usable for ADC).
  • ADC: 12-bit 500ksps ADC.
  • Digital Peripherals:
    • 2 × UART, 2 × I2C, 2 × SPI, 16 × PWM channels.
    • Timer with 4 alarms, real-time clock.
  • Programmable I/O: 2 × PIO blocks, 8 state machines in total, capable of emulating SD card and VGA interfaces.

Sensor: RC522 RFID Reader/Writer​

The MFRC522 RFID Reader/Writer is a highly versatile module designed for interfacing with RFID tags. Operating at 13.56 MHz, this sensor is integral for systems requiring identification and authentication via RFID technology. It is commonly found in applications such as access control systems, security frameworks, and attendance systems. The MFRC522 excels in environments that demand reliable communication with RFID tags to facilitate secure and efficient data exchange.

Technical Specifications and Features​
  • Integrated Circuitry: Highly integrated analog circuitry to demodulate and decode responses.
  • Antenna Connections: Buffered output drivers to connect an antenna with minimal external components.
  • Compatibility:
    • Supports ISO/IEC 14443 A/MIFARE and NTAG.
    • Typical operating distance up to 50 mm, which varies by antenna size and tuning.
  • Encryption Support: Supports MF1xxS20, MF1xxS70, and MF1xxS50 in Read/Write mode.
  • High-Speed Communication: Up to 848 kBd in ISO/IEC 14443 A mode.
  • Interfaces:
    • SPI up to 10 Mbit/s.
    • I2C-bus interface up to 400 kBd in Fast mode and up to 3400 kBd in High-speed mode.
    • RS232 Serial UART up to 1228.8 kBd.
  • Memory Management: 64-byte FIFO buffer for send-and-receive data management.
  • Power Management:
    • Software-controlled power-down.
    • Low power hard reset function.
  • Additional Features:
    • Programmable timer and I/O pins.
    • Internal oscillator for a 27.12 MHz quartz crystal.
    • CRC coprocessor for data integrity.
    • Internal self-test capabilities.

  • MFRC522 RFID Module Datasheet

Actuator: LED​

LEDs, or Light Emitting Diodes, are semiconductor devices that emit light when an electric current is applied. Known for their energy efficiency, compact size, and long life, LEDs are extensively used across various applications including electronic devices, display panels, indicators, and general lighting. Their ability to be manufactured in various colors and sizes makes LEDs exceptionally versatile for design and functional purposes in numerous projects.

Technical Specifications and Features​

  • Dice Material: Gallium Arsenide Phosphide on Gallium Phosphide (GaAsP on GaP/GaP).
  • Emitted Colors: Available in Green and Red.
  • Lens Color: Green Diffused and Red Diffused.
  • Peak Wavelength:
    • Green: 568 nm
    • Red: 587 nm
  • Viewing Angle: 45 degrees, offering focused illumination.
  • Luminous Intensity (IV):
    • Green: 15 mcd
    • Red: 14 mcd

Actuator: Buzzer​

A passive piezoelectric buzzer utilizes a piezoelectric crystal or ceramic element to create sound. When alternating voltage is applied, the piezoelectric element undergoes rapid expansion and contraction, which generates vibrations. These vibrations produce sound waves, creating audible noise. This type of buzzer is commonly used in alarms, timers, and user interface feedback due to its simplicity and effectiveness.

Technical Specifications and Features​

  • Input Voltage (Max.): 5V
  • Resistance: 42 Ω
  • Resonance Frequency: 2048 Hz
  • Body Size: 12mm x 8mm
  • Pin Pitch: 6mm
  • External Material: Plastic
  • Color: Black

References​

  • 5V Passive Buzzer Datasheet

Actuator: Servo Motor - Tower Pro SG90​

The Tower Pro SG90 9g mini servo motor is a compact and reliable actuator widely used in radio-controlled (RC) applications and projects that require precise movement control. Ideal for robotics, CNC machines, and RC car steering systems, this servo allows for detailed angular position control over a 180° range using Pulse Width Modulation (PWM). Its lightweight design and high torque capability make it suitable for a wide range of applications that demand precise positioning.

Technical Specifications and Features​

  • Operating Voltage: Typically +5V
  • Torque: 2.5 kg/cm
  • Operating Speed: 0.1 seconds per 60°
  • Gear Type: Plastic
  • Rotation: 0° to 180°
  • Weight: 9 grams
  • Included Accessories: Gear horns and screws
  • Cable Length: 15 cm

Communication Protocols​

Controller Communication​

The Raspberry Pi Pico W uses Wi-Fi and MQTT protocols to communicate with the ThingsBoard server, facilitating data exchange necessary for system operation and monitoring.

Wi-Fi Connection​

The Raspberry Pi Pico W connects to the local Wi-Fi network, providing internet access and enabling seamless interaction with external services such as the ThingsBoard server. This wireless connection is crucial for sending and receiving data in real-time, ensuring that the system remains responsive and connected.

MQTT Protocol​

MQTT (Message Queuing Telemetry Transport) is a lightweight and efficient messaging protocol designed specifically for devices with limited resources operating in potentially unreliable networks. In this system, MQTT handles the transmission of data such as access events between the Raspberry Pi Pico W and the ThingsBoard server. This protocol supports data publication from the controller to the server and subscriptions to commands or acknowledgements, enhancing the system’s reliability and security for real-time data visualization and monitoring.

Sensor Communication​

The communication between the RC522 RFID module and the Raspberry Pi Pico W controller is facilitated by the Serial Peripheral Interface (SPI) protocol.

SPI Protocol​

SPI is a synchronous serial communication interface used for short distance communication, primarily in embedded systems. In this setup, the RC522 RFID module uses SPI to transmit the identification data of RFID tags to the Raspberry Pi Pico W controller. The controller sends commands to the RFID module to initiate data transfer, and the module responds with the tag's unique identifier. This identifier is then used by the controller to determine access control decisions. The SPI protocol is chosen for its speed and reliability, ensuring that the RFID data is transferred swiftly and accurately, which is essential for maintaining system integrity and quick response times.
